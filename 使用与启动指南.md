# CSTCloud-RAG 完整使用与启动指南

最后更新：2026-06-30

> 以后每次修改项目后，都应同步更新本文件。下文中的“项目目录”是你下载或克隆 `cstcloud-rag` 后所在的文件夹。

## 最简单的打开方式

1. 打开文件资源管理器。
2. 进入你自己的 `cstcloud-rag` 项目目录。
3. 双击 `START.bat`（推荐，纯英文文件名最兼容）。`启动项目.bat` 的作用完全相同。
4. 首次启动如果弹出记事本，在 `CSTCLOUD_API_KEY=` 后填写你新生成的科技云 Token，保存并关闭记事本。
5. 等待出现“后端”和“前端”两个 PowerShell 窗口。
6. 浏览器会自动打开 `http://127.0.0.1:5173`。

以后不需要重新安装，直接双击 `START.bat` 即可。

如果双击后没有成功，可双击 `检查环境.bat`。它会告诉你缺少 Python、Node.js、依赖、`.env`，还是服务没有运行；检查完成后按任意键会直接继续启动，不会只停留在检查页面。

## 当前这次故障已经如何修复

你截图中的两个问题分别是：

1. 运行了“检查环境”，旧版本只检查、不启动，所以浏览器访问 127.0.0.1 时被拒绝。
2. 原有 `.venv` 使用 Python 3.14，但其中的 `pydantic_core` 是其他 Python 版本的二进制文件，导致后端立即退出。

现在已经修复：

- BAT 文件改成 Windows CMD 稳定识别的 ASCII + CRLF，不再出现 `'5001' 不是命令`。
- 新增纯英文入口 `START.bat`。
- 启动器会真实导入 `fastapi`、`pydantic_core` 和 `chromadb` 检查虚拟环境，而不只是判断目录存在。
- 检测到 Python 3.14 或损坏环境时，自动寻找 Python 3.11/3.12、重建 `.venv` 并安装依赖。
- “检查环境”结束后按任意键会直接进入启动流程。

本机已经实际验证：后端 `http://127.0.0.1:8000/api/health` 返回 200，前端 `http://127.0.0.1:5173` 返回 200，当前后端使用 Python 3.12。

## 第一次必须配置的 Token

Token 只能写在项目内这个不会提交到 Git 的文件中：

```text
backend\.env
```

格式：

```dotenv
CSTCLOUD_API_KEY=你新生成的科技云Token
CSTCLOUD_BASE_URL=https://uni-api.cstcloud.cn/v1
```

不要把 Token 写进前端、源代码、截图或简历。如果旧 Token 曾经出现在明文手册中，应先在科技云控制台轮换。

没有 Token 时界面仍能打开，但聊天、文档向量化和 Rerank 不能使用。

## GitHub 公开版本的隐私边界

公开仓库地址：<https://github.com/Noitexy/cstcloud-rag>

在新电脑上可使用 Git 克隆：

```powershell
git clone git@github.com:Noitexy/cstcloud-rag.git
cd cstcloud-rag
```

如果没有配置 SSH，也可以使用：

```powershell
git clone https://github.com/Noitexy/cstcloud-rag.git
cd cstcloud-rag
```

仓库只应包含源码、示例数据、测试和说明文档，不能包含以下本机内容：

- `backend/.env` 和任何真实 API Token；
- `backend/data/` 中的数据库、对话、上传文档和 Chroma 向量索引；
- `.venv`、`node_modules`、构建目录、缓存、日志及编辑器配置；
- 简历、截图、个人知识文档或其他未授权公开的资料。
- 个人面试准备材料只保存在本机，不进入公开仓库。

这些路径已经写入 `.gitignore`，安全规则见 `SECURITY.md`。公开前仍应运行密钥扫描，并在科技云控制台轮换任何曾经出现在明文文件、截图或聊天记录中的旧 Token。

当前公开版还采取了以下限制：服务默认只监听本机 `127.0.0.1`；Chroma 只作为进程内向量库使用，并用本地空实现彻底替代产品遥测客户端；在上游给出明确修复版本前，依赖固定在不受 CVE-2026-45829 影响的 `<1.0` 范围。这个作品没有用户登录、权限和租户隔离，不要直接部署到公网。

从 GitHub 克隆后，先在项目目录执行以下命令创建本地配置，真实 Token 只填写到新生成的 `.env`：

```powershell
Copy-Item backend\.env.example backend\.env
notepad backend\.env
```

然后双击 `START.bat`，或者按照后文的手动启动方式运行。

## 打开后如何导入自己的 RAG 数据

RAG 数据就是你希望模型回答时参考的专属文档，不是重新训练模型的数据。

1. 在顶部选择 Embedding 模型，初次可使用 `bge-large-zh:latest`。
2. 点击左侧“创建第一个知识库”。
3. 输入名称，例如“我的课程资料”“公司产品手册”。
4. 点击“创建并导入资料”，系统会自动选中新知识库并打开右侧资料导入面板。
5. 选择一种导入方式：
   - 点击“直接上传文件并建立索引”：可一次多选 `TXT、MD、PDF、DOCX、CSV、XLSX`，单文件最大 50 MB。
   - 点击“粘贴文本建立知识”：填写标题，粘贴网页、笔记、制度或业务资料正文，再点击“导入并建立索引”。
6. 等待文档状态变为 `ready`，并确认页面显示的知识切片数量大于 0。
7. 关闭资料面板，回到聊天区提问，在回答下方点击引用卡片检查原文。

以后要继续添加资料：先在左侧选中知识库，再点击“文档与索引管理”。上传文件和粘贴文本最终都会成为该知识库中的文档，并经过同一套解析、切片、向量化和索引流程。

第一次可上传：

```text
example-data\产品知识示例.md
```

然后提问：

```text
企业版服务可用性和故障响应时间是多少？
```

继续追问：

```text
那标准版呢？
```

第二问可以展示多轮对话和 Query Rewrite。

## 页面显示“0 个知识切片”是什么意思

说明当前没有可检索的专属资料。即使 RAG 开关打开，回答仍来自通用大模型能力，不会引用你的数据。

处理方法：创建或选择知识库 → 打开文档管理 → 上传文档 → 等待 `ready`。

## 数据保存在哪里

- 原始文件：`backend/data/uploads/`
- SQLite 元数据、文档切片和会话：`backend/data/cstcloud_rag.db`
- Chroma 向量索引：`backend/data/chroma/`

API Key 只在后端读取。但为了生成向量和回答问题，相关文档切片会通过后端发送给所选 CSTCloud Embedding/Chat 模型。不要上传不允许发送给外部云模型的敏感资料。

## 如何停止项目

启动后会出现两个窗口：

- `CSTCloud-RAG 后端（关闭此窗口即停止）`
- `CSTCloud-RAG 前端（关闭此窗口即停止）`

分别在两个窗口按 `Ctrl+C`，或者直接关闭两个窗口。

## 修改代码后如何重新打开

1. 关闭前端和后端窗口。
2. 再次双击 `START.bat`。

开发模式已经开启自动重载：一般保存后端 Python 或前端 Vue/CSS 文件后，页面会自动更新；如果没有更新，按 `Ctrl+F5` 强制刷新浏览器。

## 手动启动方式

如果一键脚本失败，可手动打开两个 PowerShell。

后端窗口：

```powershell
cd <你的项目目录>\cstcloud-rag\backend
.\.venv\Scripts\Activate.ps1
uvicorn app.main:app --reload --port 8000
```

前端窗口：

```powershell
cd <你的项目目录>\cstcloud-rag\frontend
npm run dev
```

浏览器访问：`http://127.0.0.1:5173`。

## 全新电脑首次安装

建议安装：

- Python 3.11 或 3.12
- Node.js 20 LTS

一键脚本会自动创建 `.venv` 并执行 `pip install -r requirements.txt`，也会在缺少前端依赖时执行 `npm install`。首次安装需要网络，之后启动不再重复安装。

## 常见问题

### 双击后窗口一闪而过

双击 `检查环境.bat` 查看具体原因。通常是没有安装 Python 或 Node.js。

### 页面提示 503 或未配置 API Key

检查 `backend/.env` 是否存在，并确认 `CSTCLOUD_API_KEY=` 后不是空值。保存后重启后端。

### 页面打不开

确认两个服务窗口没有红色错误。分别访问：

- `http://127.0.0.1:8000/api/health`
- `http://127.0.0.1:5173`

### 上传后文档状态 failed

查看文档卡片上的错误信息。常见原因是 Token 无效、账号未授权 Embedding 模型、文件没有可解析文本，或网络请求失败。

### 扫描版 PDF 没有文字

当前普通 PDF 解析器不能识别图片文字。先使用 OCR 转成可搜索 PDF，或把文字保存为 TXT/Markdown 后上传。

### “创建知识空间”只显示在左侧一小块

该问题已经在 2026-06-29 修复：弹窗现在挂载到页面最上层并居中显示，不再被左侧栏裁切。如果浏览器仍显示旧样式，按 `Ctrl+F5` 强制刷新；仍未变化时，关闭前后端窗口后重新双击 `START.bat`。

### 更换 Embedding 模型后怎么办

不同模型向量维度可能不同。界面提示模型不一致时，点击右侧“重建”，不要把新旧模型向量写入同一个索引。
